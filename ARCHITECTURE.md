## ğŸ¨ State Management

Using **BLoC (Cubit)** pattern:
- **Separation of Concerns**: UI doesn't know about data sources
- **Reactive**: UI rebuilds automatically when state changes
- **Testable**: Logic can be tested independently
- **Scalable**: Easy to add new features

## ğŸ“¦ Key Components

### Data Models
- **AzkarModel**: Represents a single Azkar with title, text, repeat count, and counter
- **AzkarCategoryModel**: Groups multiple Azkar (e.g., Morning Azkar, Evening Azkar)

### State Management
- **Cubit**: Manages application state and business logic
- **States**: Represent different stages (loading, loaded, error)

### UI Components
- **AzkarScreen**: Main screen with header, progress, and list
- **AzkarCard**: Reusable card showing individual Azkar

## ğŸ”„ Benefits of This Architecture

1. **Separation of Concerns**: Each layer has a specific responsibility
2. **Testability**: Easy to write unit tests for each layer
3. **Maintainability**: Changes in one layer don't affect others
4. **Scalability**: Easy to add new features (Evening Azkar, Favorites, etc.)
5. **Reusability**: Components can be reused across the app
